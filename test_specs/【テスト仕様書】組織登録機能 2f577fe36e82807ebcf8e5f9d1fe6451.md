## 10. 組織登録機能 (/register-organization)

### 基本情報

- **URL**: `/register-organization`
- **コンポーネント**: `src/components/environments/RegisterOrganization/`
- **機能概要**: 新規に利用を開始する際の組織（カンパニー）情報登録画面。会社名や住所などを入力し、利用環境を構築する。

### テスト項目

### 1. 登録フォーム機能テスト (REG)

| 項目 ID | 操作/確認内容 | 期待結果 | 実施者 | 実施日 | 結果 | 備考 |
| --- | --- | --- | --- | --- | --- | --- |
| REG-FORM-001 | ページアクセス | 組織登録画面へアクセス。<br>URL: `/register-organization` | 「オーガニゼーションの作成」フォームが表示されること。<br>（`organization:create` 権限が必要） |  |  |  |
| REG-FORM-002 | 必須項目入力 | 会社名、住所等の必須項目を入力して「送信」をクリック。 | 処理が成功し、「オーガニゼーションを作成しました」という完了メッセージと組織 ID が表示されること。 |  |  |  |
| REG-FORM-003 | バリデーション | 必須項目（会社名など）を空欄にして送信。 | エラーメッセージが表示され、送信できないこと。 |  |  |  |
| REG-FORM-004 | 完了後の動作 | 作成完了画面の「ログアウトする」ボタンをクリック。 | ログアウト処理が実行され、ログイン画面へ遷移すること。 |  |  |  |

### 2. 権限テスト (AUTH)

| 項目 ID | 確認内容 | 期待結果 | 実施者 | 実施日 | 結果 | 備考 |
| --- | --- | --- | --- | --- | --- | --- |
| REG-AUTH-001 | 権限なしアクセス | `organization:create` 権限を持たないユーザーでアクセス。 | アクセス拒否（403）ページ、または何も表示されない（PermissionCheck 動作）こと。 |  |  |  |
